# Wazak PRD

## 1. 개요

Wazak은 사용자가 "말랑이" 캐릭터를 등록하고, 상호작용하고, 공유할 수 있는 macOS 플로팅 컴패니언 앱입니다.

앱은 항상 최상단에 작은 캐릭터를 띄워줍니다. 캐릭터를 클릭하면 랜덤 소리가 재생됩니다. 사용자는 이미지와 소리로 커스텀 말랑이를 등록하고, Supabase를 통해 동기화하며, 마켓플레이스에 공유하거나 다운로드할 수 있습니다.

## 2. 목표

- 사용자가 Mac 화면에 작은 인터랙티브 말랑이를 띄울 수 있게 한다.
- 사용자가 이미지, 이름, 여러 소리로 자신만의 말랑이를 등록할 수 있게 한다.
- 사용자가 말랑이를 클릭하면 랜덤 소리가 재생되게 한다.
- 사용자가 Supabase를 통해 말랑이 메타데이터와 에셋을 동기화할 수 있게 한다.
- 사용자가 선택한 말랑이를 마켓플레이스에 업로드할 수 있게 한다.
- 사용자가 마켓플레이스 말랑이를 탐색하고 자신의 로컬 목록으로 다운로드할 수 있게 한다.

## 3. 비목표

- 결제, 크리에이터 수익화, 유료 다운로드.
- 공개 신고, 댓글, 좋아요, 평점, 콘텐츠 관리.
- 정식 Mac App Store 출시 절차.
- 프로덕션 수준의 계정 어뷰징 방지.
- 모든 이미지에 대한 완벽한 로컬 배경 제거.

## 4. 현재 MVP 범위

### 4.1 플로팅 말랑이

앱은 투명한 플로팅 오버레이 형태의 macOS accessory 스타일 앱으로 실행됩니다.

현재 동작:

- 말랑이 이미지를 표시한다.
- 이미지 아래에 말랑이 이름을 뱃지로 표시한다.
- 앱 영역에 호버 시 이전/다음 화살표와 설정 아이콘이 나타난다.
- 말랑이를 클릭하면 등록된 소리 중 하나가 랜덤 재생된다.
- 클릭 시 작은 탭 애니메이션이 실행된다.
- 기본 번들 말랑이는 `사과 슬랑이`이다.
- 이전 플레이스홀더 말랑이인 `젤리 말랑이`와 `밤톨 말랑이`는 제거됐다.

### 4.2 로컬 말랑이 등록

사용자는 설정 창을 열어 커스텀 말랑이를 등록할 수 있습니다.

필드 및 액션:

- 말랑이 이름 입력.
- 이미지 선택.
- 선택한 이미지 배경 제거 시도.
- 이미지 미리보기.
- 여러 소리 선택.
- 등록된 소리 목록.
- 소리별 삭제.
- 저장 버튼 (스피너 포함).
- 기존 말랑이 선택 후 편집 가능.
- 등록된 말랑이 삭제 가능.

동작:

- 저장 후, 새로 생성하거나 편집한 말랑이가 플로팅 오버레이에 선택된 말랑이가 된다.
- 설정 창을 다시 열면 폼이 새 등록 상태로 초기화된다.
- 등록된 이미지 표시 이름은 내부 업로드 파일명과 별도로 유지된다.
- 등록된 소리 표시 이름은 내부 업로드 파일명과 별도로 유지된다.

### 4.3 Supabase 동기화

Supabase는 클라우드 동기화 및 에셋 저장에 사용됩니다.

현재 환경 변수:

```txt
SUPABASE_URL
SUPABASE_PUBLISHABLE_KEY
```

로컬 `.env` 파일은 개발용으로 사용되며 Git에서 제외됩니다.

현재 동기화 동작:

- 말랑이 메타데이터는 `public.malangis`에 저장된다.
- 이미지와 소리는 Supabase Storage 버킷 `malangi-assets`에 업로드된다.
- Storage 오브젝트 키는 UUID 기반 이름으로 안전하게 생성된다.
- 원본 이미지와 소리 이름은 표시용 메타데이터로 저장된다.
- Supabase 환경 변수가 없으면 로컬 전용 모드로 실행된다.

### 4.4 마켓플레이스 MVP

설정 창은 두 가지 모드를 포함합니다:

```txt
내 말랑이 / 마켓플레이스
```

마켓플레이스 동작:

- 사용자는 Supabase Auth를 통해 이메일/비밀번호로 로그인 또는 회원가입할 수 있다.
- 로그인한 사용자는 선택한 로컬 말랑이를 마켓플레이스에 업로드할 수 있다.
- 사용자는 공개 마켓플레이스 말랑이를 새로고침하고 탐색할 수 있다.
- 사용자는 마켓플레이스 말랑이를 자신의 로컬 목록으로 다운로드할 수 있다.
- 다운로드한 말랑이는 즉시 플로팅 오버레이에서 사용 가능하다.

## 5. Supabase 데이터 모델

### 5.1 `public.malangis`

목적: 사용자 앱 동기화 / 로컬 등록 말랑이 메타데이터.

주요 필드:

```txt
id uuid
name text
image_path text
image_file_name text
sound_paths text[]
sound_file_names text[]
created_at timestamptz
updated_at timestamptz
```

현재 주의사항: MVP 수준이며 허용적인 공개 정책 사용 중. 프로덕션 전 수정 필요 — §10 참고.

### 5.2 `public.marketplace_malangis`

목적: 공개 마켓플레이스 목록.

필드:

```txt
id uuid
owner_id uuid references auth.users(id)
owner_email text
name text
image_url text
image_file_name text
sound_urls text[]
sound_file_names text[]
description text
tags text[]
downloads_count integer
is_public boolean
created_at timestamptz
updated_at timestamptz
```

RLS 정책 의도:

- 공개 사용자는 공개 목록을 읽을 수 있다.
- 인증된 사용자는 자신이 소유한 마켓플레이스 목록을 삽입할 수 있다.
- 인증된 사용자는 자신의 목록만 수정/삭제할 수 있다.

### 5.3 Storage

버킷:

```txt
malangi-assets
```

사용:

- 말랑이 이미지를 저장한다.
- 말랑이 소리를 저장한다.
- 공개 URL이 데이터베이스 행에 저장된다.
- 내부 Storage 키는 안전하게 생성되며 표시 이름과 일치할 필요가 없다.

## 6. 인증 모델

현재 구현:

- Supabase 이메일/비밀번호 로그인.
- Supabase 이메일/비밀번호 회원가입.
- 세션은 `UserDefaults`에 로컬 저장된다.
- 인증된 세션은 마켓플레이스 업로드에 사용된다.

미해결 문제: 회원가입이 너무 개방적이며 다중 계정 생성이 가능하다. 이메일 인증 동작은 Supabase 대시보드 설정에 의존한다. 앱은 현재 초대 코드·계정 승인·업로드 승인을 구현하지 않는다. 대응 방안은 §11 마일스톤 1 참고.

## 7. 사용자 플로우

### 7.1 로컬 말랑이 등록

```txt
설정 열기
내 말랑이 선택
새 말랑이 클릭
이름 입력
이미지 선택
소리 하나 이상 선택
확인 클릭
플로팅 오버레이가 저장된 말랑이로 전환됨
```

### 7.2 로컬 말랑이 편집

```txt
설정 열기
내 말랑이 선택
등록된 말랑이 선택
이름/이미지/소리 변경
확인 클릭
플로팅 오버레이가 업데이트된 말랑이로 전환됨
```

### 7.3 마켓플레이스 업로드

```txt
설정 열기
내 말랑이 선택
등록된 말랑이 선택
마켓플레이스로 전환
로그인
마켓플레이스에 올리기 클릭
앱이 마켓플레이스 행과 에셋을 업로드함
마켓플레이스 목록 새로고침됨
```

### 7.4 마켓플레이스 다운로드

```txt
설정 열기
마켓플레이스 선택
목록 새로고침
마켓플레이스 말랑이 선택
다운로드 클릭
말랑이가 로컬 목록에 추가됨
플로팅 오버레이가 다운로드된 말랑이로 전환됨
```

## 8. UX 요구사항

- 플로팅 오버레이는 가볍고 방해가 되지 않는 느낌이어야 한다.
- 설정은 실용적이고 간결해야 하며, 마케팅 페이지처럼 느껴지지 않아야 한다.
- 사용자는 항상 어떤 말랑이가 선택되어 있는지 알 수 있어야 한다.
- 이미지와 소리 파일 표시 이름은 내부 Storage 오브젝트 이름이 아닌 사용자 기대에 맞아야 한다.
- 마켓플레이스 업로드는 로그인이 필요함을 명확히 해야 한다.
- 마켓플레이스 다운로드는 마찰이 적어야 한다.

## 9. 기술 요구사항

- macOS 13+.
- 현재 Swift Package 실행파일.
- AppKit UI.
- AVFoundation 소리 재생.
- `URLSession`을 통한 Supabase REST API.
- REST 엔드포인트를 통한 Supabase Storage 업로드.
- REST 엔드포인트를 통한 Supabase Auth.

## 10. 알려진 문제 및 리스크

- 앱이 아직 서명된 `.app`으로 패키징되지 않았다.
- `.build` 아래의 빌드 아티팩트가 이전 커밋에서 Git에 추적되고 있다.
- Supabase `public.malangis` 정책이 프로덕션에 비해 너무 허용적이다.
- 마켓플레이스 회원가입이 프로덕션에 비해 너무 개방적이다.
- 이메일 인증 동작이 Supabase 대시보드 설정에 따라 달라진다.
- 배경 제거는 최선 시도이며 어려운 이미지에서 실패할 수 있다.
- 마켓플레이스 콘텐츠 관리나 어뷰징 제어가 없다.
- 다운로드 수 증가 플로우가 아직 없다.
- 소유자가 마켓플레이스 목록을 관리하는 UI가 아직 없다.

## 11. 다음 마일스톤 제안

### 마일스톤 1: 마켓플레이스 안전

- 앱 내 회원가입 숨기기 또는 제거.
- 로그인만 유지.
- Supabase에서 테스트 사용자 수동 생성.
- 명확한 로그인/오류 메시지 추가.

### 마일스톤 2: 마켓플레이스 완성도

- 마켓플레이스 목록에 이미지 썸네일 추가.
- 선택한 마켓플레이스 말랑이의 상세 보기 추가.
- 이메일 원문 대신 크리에이터 닉네임 추가.
- 다운로드 수 증가 추가.

### 마일스톤 3: 계정 및 업로드 제어

- `profiles` 테이블 추가.
- `can_upload` 플래그 추가.
- 승인된 업로더로 마켓플레이스 insert 정책 제한.
- 마켓플레이스 목록 소유자 삭제/편집 추가.

### 마일스톤 4: 배포

- `.app` 번들 패키징 스크립트 생성.
- 앱 아이콘 추가.
- 서명 및 공증 플로우 추가.
- DMG 또는 zip 아티팩트 빌드.

## 12. 성공 지표

MVP 성공:

- 사용자가 이미지와 소리로 말랑이를 만들 수 있다.
- 사용자가 말랑이를 클릭하면 랜덤 소리가 들린다.
- 사용자가 로그인 후 말랑이를 마켓플레이스에 업로드할 수 있다.
- 사용자가 마켓플레이스 말랑이를 다운로드해 즉시 사용할 수 있다.

미래 지표:

- 공개된 말랑이 수.
- 다운로드 수.
- 활성 사용자 수.
- 업로드 실패율.
- 다운로드 후 사용 전환율.
